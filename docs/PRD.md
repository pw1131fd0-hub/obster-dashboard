# OpenClaw 小龍蝦系統監控儀表板 PRD

## 1. 產品願景

OpenClaw Dashboard 是一個本地 Web UI 儀表板，專為在 VPS（srv1318420）上監控 OpenClaw 小龍蝦系統的整體運行狀態而設計。系統管理員透過瀏覽器即可即時掌握開發任務狀態、Cron Job 執行、Agent 健康度與執行日誌，無需 SSH 登入伺服器或手動查詢日誌。

## 2. User Stories

- **身為系統管理員**，我希望在一個統一的 dashboard 看到所有專案的開發狀態，以便快速識別哪些專案正在落後或被阻塞。
- **身為 DevOps 工程師**，我希望監控所有 Cron Job 的執行狀態和最後運行時間，以便在任務失敗時立即發現並處理。
- **身為系統維護者**，我希望即時知道每個 Telegram Agent 的健康狀態（最後回應時間），以便在 Agent 無回應超過 30 分鐘時發送警報。
- **身為開發者**，我希望查看最近 20 筆執行日誌，以便追蹤系統行為和排查問題。

## 3. 功能分級

### P0（核心必備）
- 四大面板：開發任務狀態、Cron Job 監控、Agent 健康度、執行 Log
- FastAPI 後端 + React 前端
- 每 30 秒自動刷新資料
- 暗色主題 UI
- Docker 部署

### P1（重要）
- 讀取 docs/.dev_status.json 顯示專案品質分數
- 顯示每個專案的當前 stage（prd/dev/test/security）
- 顯示 blocking_errors 列表
- Cron Job 最後執行時間顯示
- Agent 無回應 30 分鐘以上標記為異常

### P2（增強）
- 日誌檔案點擊展開查看詳細內容
- 時間範圍篩選器
- 健康度異常的視覺化警示（紅色標記）

## 4. 非功能需求

- **效能**：頁面載入時間 < 3 秒，API 回應時間 < 500ms
- **可用性**：Docker container 自動重啟，掛載 volume 保存資料
- **擴展性**：模組化面板設計，未來可新增面板類型
- **安全性**：僅本地存取（localhost），不對外暴露通訊埠；無敏感資料寫入日誌

## 5. 技術選型

| 層面 | 技術 | 選擇理由 |
|------|------|----------|
| 前端框架 | React 18 + Vite | 快速開發、HMR 熱更新、龐大生態系 |
| UI 组件 | 自建元件（Tailwind CSS） | 無需額外依賴，完整控制暗色主題 |
| 後端框架 | FastAPI (Python 3.11) | 非同步效能佳、自動化 OpenAPI 文件、輕量 |
| 資料格式 | JSON | 與現有 .dev_status.json 格式一致 |
| 日誌讀取 | 直接檔案讀取 (pathlib) | 無需資料庫，純本地檔案系統操作 |
| 部署方式 | Docker + docker-compose | 一鍵部署、環境一致性 |
| 容器執行 | systemd 管理的 Docker container | VPS 長期運行、auto-restart |

## 7. 成功指標

1. **可用性**：Dashboard 正常運行時間 > 99%（每月停機時間 < 7 小時）
2. **效能**：首屏載入時間 < 2 秒，資料刷新延遲 < 1 秒（從讀取到顯示）
3. **準確性**：Agent 健康度判斷準確率 100%（以 Telegram Bot API 的 message 時間為準）
4. **覆蓋率**：100% 的專案、任務、Agent、執行日誌都能在 dashboard 中呈現
